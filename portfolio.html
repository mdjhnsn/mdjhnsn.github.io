  • Actionable outputs — delivered survival probabilities marketing could act on

Data Lakehouse Architecture

Fancy (Fintech) | Aug 2025 – Jan 2026

The Challenge

Fancy's analytics queries were running directly against the production database, creating downtime risks during peak reporting periods. The organization needed a scalable data infrastructure that decoupled analytics from production while providing a single source of truth.

My Approach

  • Architecture: Designed and deployed Data Lakehouse using AWS DMS for replication, S3 for storage, Glue for cataloging, and Athena for querying
  • Infrastructure as Code: Used Terraform for all provisioning to ensure repeatability and environment consistency
  • Scale: Replicated 8 core schemas and 124 tables, creating single source of truth for 2,500+ merchants
  • Governance: Established data quality checks and access controls

Results

  • Eliminated production downtime risk — 100% of analytical queries moved off operational RDS
  • 100% system availability — Finance, Product, and Marketing teams gained reliable 24/7 access
  • 90% reduction in environment setup time — Terraform automation enabled consistent deployments

Tech Stack

AWS DMS S3 Glue Athena Terraform Infrastructure as Code

LLM Entity Resolution & RAG Application

Independent Project | 2024

The Challenge

Duplicate detection in large catalogs (e.g., licensed Starbucks store items) required manual review. Building a system that could identify near-duplicates with high accuracy while ranking retrieved documents for human review.

My Approach

  • Entity Resolution: Measured catalog item similarity using product description vector embeddings with Sentence-Transformers and PyTorch
  • RAG Application: Built Retrieval-Augmented Generation tool to dynamically retrieve and re-rank 100K+ documents
  • Evaluation: Achieved 98% accuracy in duplicate detection through careful threshold tuning

Results

  • 98% accuracy in duplicate detection
  • 30 hours monthly saved in manual review time
  • Scalable architecture — handles 100K+ documents with sub-second retrieval
